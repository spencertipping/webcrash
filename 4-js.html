Introduction to JavaScript

Spencer Tipping

First, pop open this useful JavaScript console.

Here's some JavaScript

var x = 5;
if (x > 2) {
  alert ("Greater than 2!");
} else if (x < 0) {
  alert ('Less than zero');
}
Looks a lot like Java. (Not an accident, by the way.)

Where JavaScript == Java

  • Operators: +, -, *, /, ^, &, |, !, %, <, >, ==, !=, ?:, etc.
  • Dot-notation: object.method()
  • if, for, while, do-while, switch
  • Line (//) and block (/* ... */) comments
  • Numeric literals, boolean literals, and null
  • These all behave exactly the way you would expect them to behave in Java, and have the same syntax.

Where JavaScript is much cooler

  • Regular expressions: /f.o?[bar]/gi
  • Array literals: [1, 'foo', false]
  • Object literals: {foo: 'bar', length: 3}

Object literals?

  • First do this: var o = {foo: 'bar', length: 3};
  • And now try this: o.foo
  • And this: o.length
  • The dot-notation means "grab this member of an object".

And the best part:

Function literals

var f = function (x) {
  return x + 1;
};
f(5)            // returns 6
f('foo')        // returns 'foo1'

Prototypes

  • Let you do OOP
  • Much more powerful than classes
  • Can be generated programmatically (though doing this can get you fired)
Let's build this Java class in JavaScript: 
class Point {
  public Point (double x, double y) {
    this.x = x;
    this.y = y;
  }
  public double x;
  public double y;
  public double distance() {
    return Math.sqrt (x*x + y*y);
  }
}
First we create a constructor: 
var Point = function (x, y) {
  this.x = x;
  this.y = y;
};
Then we add the distance method: 
Point.prototype.distance = function () {
  return Math.sqrt (this.x * this.x + this.y * this.y);
};
And that's it -- we're done.
Here's how to use it: 
var p = new Point(3, 4);
p.x           // returns 3
p.y           // returns 4
p.distance()  // returns 5

How Prototypes Work

  • Each object has its own list of members
  • When you ask for a member that isn't there, the object searches its prototype
  • If the member isn't there either, then you get undefined

How Prototypes Work

var f = function () {return 'foo'};
({toString: f}).toString()     // returns 'foo'
({foo: 'bar'}).toString()      // returns '[object Object]'
({foo: 'bar'}).nonexistent     // returns undefined
