Prototypes

  • Let you do OOP
  • Much more powerful than classes
  • Can be generated programmatically (though doing this can get you fired)
Let's build this Java class in JavaScript: 
class Point {
  public Point (double x, double y) {
    this.x = x;
    this.y = y;
  }
  public double x;
  public double y;
  public double distance() {
    return Math.sqrt (x*x + y*y);
  }
}
First we create a constructor: 
var Point = function (x, y) {
  this.x = x;
  this.y = y;
};
Then we add the distance method: 
Point.prototype.distance = function () {
  return Math.sqrt (this.x * this.x + this.y * this.y);
};
And that's it -- we're done.
Here's how to use it: 
var p = new Point(3, 4);
p.x           // returns 3
p.y           // returns 4
p.distance()  // returns 5

How Prototypes Work

  • Each object has its own list of members
  • When you ask for a member that isn't there, the object searches its prototype
  • If the member isn't there either, then you get undefined

How Prototypes Work

({foo: 'bar'}).nonexistent     // returns undefined
