How CSS works

  • selectors pick a bunch of elements from the document
  • rules change the appearance of those elements
Pop open this editor to try it out.

Basic selectors

  • p, div, ...: match elements of that type
  • .class: match elements with a particular "class" attribute
  • #id: match the element with a particular "id" attribute

Combining selectors

  • p#foo: matches the paragraph whose ID is "foo"
  • p > div: matches any div that is a direct child of a paragraph
  • p div: matches any div that is a descendant of a paragraph
  • p, div: matches all paragraphs and divs

Common rules

  • font-family, font-size, font-weight, ...: changes font properties
  • color: sets text color (e.g. red, green, #880000, #080)
  • background: sets background color, image, or pattern
  • border: puts a border on something
  • margin: puts space around an element
  • padding: puts space inside an element
  • width, height: sets the size of a block element

Big Important CSS Concept #1

Inline vs. block

This is determined by the display property.

Valid values include block, inline, inline-block, and none (among others).

Big Important CSS Concept #2

Positioning

This is determined by the position property.

Valid values include static, relative, absolute, and fixed.

Positioning rules

  • Everything is static by default
  • Non-static elements are offset parents
  • left and top are relative to the offset parent
  • Absolute elements don't displace anything
